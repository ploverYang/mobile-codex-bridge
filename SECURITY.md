# 安全说明

## 权限模型

手机 PWA 会读取并使用电脑端 Codex 当前允许的访问等级，用户可在新建任务和续聊前选择；默认仍为“完全访问”。选择完全访问时，已配对设备可以让 Codex 执行命令并修改电脑文件；项目白名单限制手机可选择的工作目录，但不能把完全访问本身视为操作系统安全沙箱。

只为自己控制并信任的设备配对。丢失手机、怀疑会话泄露或不再使用时，应立即在手机退出配对，或停止 Bridge 并清理受影响会话。

## 网络边界

- 默认监听 `127.0.0.1`。
- 跨设备访问推荐 Tailscale Serve 和 tailnet ACL。
- 不要把 Codex App Server、Bridge 管理 API 或 `data/` 目录公开到互联网。
- 局域网 `0.0.0.0` 模式缺少自动 HTTPS，只适合用户明确接受风险的临时场景。

## 凭据和本地数据

- Bridge 不保存 ChatGPT 密码、Cookie 或 OpenAI API Key。
- 一次性配对码默认有效 7 天，成功使用后立即失效。
- 会话令牌随机生成并哈希存储；手机退出会撤销服务端会话。
- `data/state.json` 包含管理和会话安全状态，禁止读取、分享或提交。
- `data/tasks.json` 可能包含任务标题和线程 ID，同样应视为私人数据。
- `storage.persistOutputs` 默认关闭；完整对话仍保存在 Codex 自己的本地任务历史中。

## 微信边界

微信公众号适配器目前不支持 AES 安全模式、持久队列和完整异步回执。生产公众号部署前必须补齐这些能力，并且反向代理只能公开 `/wechat/callback`，不能顺带公开 PWA 和管理 API。

## 报告问题

公开提交 Issue 前请移除用户名、本机路径、项目名称、线程 ID、tailnet 域名、日志中的提示词和所有令牌。安全漏洞不要附带可用凭据或真实私人数据。
