# Changelog

## 0.23.0

- 手机首页压缩为一屏任务面板：配对与解除入口改成页面最顶部的小状态条，并移除与“电脑在线”重复的 Codex 在线状态。
- 首页移除右上角重复的新建任务加号和完全访问说明。
- 表单顺序调整为项目、任务描述、执行配置、发送；语音提示收进任务描述框底部。

## 0.22.0

- 任务详情页改为会话优先布局，标题压缩为单行，并移除项目名、线程编号、轮次摘要、提示和底栏。
- 访问等级、模型和推理深度改为输入框下方的紧凑工具栏，仅展示当前值，点击时使用系统选择器展开选项。
- “电脑端打开”和运行中任务的停止操作压缩为工具栏图标；归档入口仅保留在任务列表，详情页为对话内容释放更多纵向空间。

## 0.21.0

- 最近任务改为高密度紧凑卡片，移除输出预览、线程编号、列表续聊和文字操作区；归档、恢复与停止改为单个图标按钮。
- 首页和任务详情页动态读取电脑端 Codex 的访问等级、可用模型及各模型支持的推理深度，并将选择应用到新建任务和后续追问。
- 访问等级直接使用 Codex `permissionProfile/list` 返回的原生权限配置，项目级限制与电脑端保持一致。
- 详情追问输入框获得焦点时自动放大，便于输入和校对较长内容。

## 0.20.0

- 右上角手动刷新现在会强制同步电脑连接、任务列表、当前任务状态和完整详情，不再受自动轮询的前台条件限制。
- 刷新按钮会明确显示同步中、成功或失败状态，并在详情标题区域显示结果反馈。
- 所有 API 刷新请求显式绕过浏览器缓存，确保读取服务端最新内容。

## 0.19.2

- 将高频只读轮询与写操作限流分离，并让健康检查不计入限额，避免正常刷新触发 429。
- 详情请求超过 20 秒会自动释放；手动刷新和重新进入详情可取消卡住的旧请求并立即重试。
- 网络错误不再清空已经显示的会话内容，持久化线程完成后会自动修正遗漏的本地完成状态。

## 0.19.1

- 修复仅通过 30 天 HttpOnly Cookie 恢复登录时，任务列表因缺少本地 Bearer token 而不加载的问题。
- Cookie 恢复的会话现在也能从手机端正常退出并撤销。

## 0.19.0

- 修复任务详情把运行时增量与持久化历史合并后重复展示同一条 Codex 回复。
- 第二轮及后续轮次完成时始终强制桌面端重新加载线程，包括 Bridge 重启后的续聊。
- 配对成功后同时保存 30 天 HttpOnly 会话 Cookie，手机本地存储丢失时可自动恢复登录状态。

## 0.18.0

- 对缺少 App Server rollout 的旧任务提供本地归档兜底，不再阻止手机端归档。
- 明确显示未同步到电脑端 Codex 的本地归档状态，并支持本地恢复。

## 0.17.0

- 修复 ES Module `.mjs` 文件被错误返回为二进制类型，导致手机 PWA 停留在“连接中”的问题。

## 0.16.0

- 归档任务会立即从“全部”及其他非归档筛选中移除，只在“已归档”中展示。
- 防止较早的任务列表轮询响应覆盖刚完成归档的最新状态。

## 0.15.0

- 新增 Windows 一站式安装向导。
- 新增面向用户和 AI 的 `doctor` / `doctor --json` 环境诊断。
- 重写公开发布 README，新增 AI 安装、故障排查、安全与开发文档。
- 明确一次性配对码默认保持 7 天且成功使用后立即失效。

## 0.14.0

- 手机任务中心增加归档筛选，归档与恢复同步到 Codex Desktop。

## 0.11.0

- 手机新建和续聊任务默认使用完全访问及免审批策略。

## 0.7.0

- 新增真实多轮任务详情、运行过程和桌面线程刷新。
